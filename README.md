# Скоринг зрелости QA-команды

Одностраничное клиентское приложение для заполнения чек-листа зрелости QA-команды
(шаблон v18.06.2026). Данные не покидают браузер, сервер не нужен.

## Возможности

* Заполнение чек-листа онлайн: название проекта и дата оценки (по умолчанию сегодня).
* Кнопка «Выбрать все» для каждого блока внутри уровня.
* Радарная диаграмма по трем блокам: Разработка автотестов, Разработка тест-кейсов,
  Выполнение внутреннего тестирования релиза.
* Детальные карточки по каждому блоку: процент выполнения и статус по каждому из 5 уровней.
* Выгрузка результата в JSON и загрузка ранее сохраненного JSON.
* Печать страницы (в печатной версии кнопки скрыты).

## Логика расчета

### Итоговая зрелость команды

* Уровень считается достигнутым, если отмечено не менее 80% его пунктов, при этом
  пункты всех трех блоков на этом уровне считаются одним пулом.
* Перепрыгнуть уровень нельзя: итоговый уровень равен максимальному уровню, для которого
  достигнуты все уровни начиная с первого. Если первый уровень не достигнут, итог равен 0.
* Итог не зависит от того, как выполненные пункты распределены между блоками. Отставание
  одного блока можно компенсировать другими, пока пул уровня держит 80%.

### Уровень отдельного блока

Радар и карточки показывают зрелость каждого блока отдельно. Там те же 80% и та же
последовательность, но считаются только пункты самого блока. Поэтому итог команды и
уровень отдельного блока могут не совпадать, это ожидаемо.

Всего в чек-листе 41 пункт. Распределение по уровням и блокам:

| Уровень | Автотесты | Тест-кейсы | Тестирование релиза | Пул уровня | Нужно для 80% |
| --- | --- | --- | --- | --- | --- |
| 1 (начальный) | 3 | 2 | 4 | 9 | 8 |
| 2 (базовый) | 3 | 4 | 7 | 14 | 12 |
| 3 (развитый) | 5 | 5 | 2 | 12 | 10 |
| 4 (системный) | 1 | 1 | 1 | 3 | 3 |
| 5 (активный) | 1 | 1 | 1 | 3 | 3 |

На уровнях 4 и 5 пул составляет 3 пункта, поэтому 80% там требуют отметить все три.

## Публикация на GitHub Pages

Дополнительная сборка и настройка не требуются, это статические файлы.

1. Запушить репозиторий на GitHub.
2. Settings -> Pages -> Source: Deploy from a branch.
3. Branch: `main`, папка `/ (root)`, затем Save.

Через минуту сайт будет доступен по адресу `https://<username>.github.io/<repo>/`.

## Локальный запуск

Достаточно открыть `index.html` в браузере. Либо поднять локальный сервер:

```bash
python3 -m http.server 8000
```

и открыть `http://localhost:8000`.

## Файлы

| Файл | Назначение |
| --- | --- |
| `index.html` | Разметка страницы |
| `styles.css` | Стили, включая версию для печати |
| `checklist-data.js` | Данные чек-листа, формулировки перенесены из шаблона без изменений |
| `app.js` | Расчет метрик, диаграмма, экспорт и импорт JSON |

## Формат JSON

```json
{
  "schema": "qa-team-scoring",
  "schemaVersion": 1,
  "templateVersion": "18.06.2026",
  "exportedAt": "2026-07-15T10:00:00.000Z",
  "projectName": "Портал технадзора",
  "assessmentDate": "2026-07-15",
  "answers": { "L1-at-0": true },
  "results": {
    "overallMaturity": 2,
    "overallLevels": [
      { "level": 1, "checkedItems": 9, "totalItems": 9, "percent": 100, "achieved": true }
    ],
    "blocks": []
  }
}
```

`overallLevels` это пул по всем блокам на каждом уровне, по нему считается
`overallMaturity`. В `blocks` лежит детализация по каждому блоку отдельно.

При загрузке читаются `projectName`, `assessmentDate` и `answers`. Раздел `results`
пересчитывается заново, поэтому его правка вручную ни на что не влияет. Идентификаторы
пунктов имеют вид `L<уровень>-<блок>-<номер>`, где блок это `at`, `tc` или `rt`.
Пункты, которых нет в текущем шаблоне, игнорируются с предупреждением.

## Зависимости

[Chart.js 4.4.1](https://www.chartjs.org/) подключается с CDN jsDelivr. Если CDN
недоступен, страница продолжает работать: вместо диаграммы показывается сообщение,
а уровни остаются видны в карточках блоков.
